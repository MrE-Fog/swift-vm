Instructions for AAT

  1. Connect AAT to ethernet
  2. Update DNS record A mail.[YOURDOMAIN].com to your EXTERNAL_IP
  3. Update DNS record MX mail.[YOURDOMAIN].com to host [YOURDOMAIN].com
  4. Create DNS record TXT for host [YOURDOMAIN].com with value "v=spf1 mx ~all" (without quotes)
  5. Forward ports 80,25,465,587,143,993,110,995,443 to AAT
  6. With Bitvise SSH to AAT using username 'ubuntu' and password 'admin' without quotes
  7. Run su with password 'admin' to become root user
  8. Run "./shell.sh berry123 NEW_ROOT_LOGIN_PASSWORD [YOURDOMAIN].com" without quotes in /home/ubuntu/
  9. Run "./cert.sh [YOURDOMAIN].com" without quotes in /home/ubuntu/
  10. shell.sh ran successfully when the last line reads "Success"
  11. Login to your admin account at https://mail.[YOURDOMAIN].com/postfixadmin/public/login.php with username admin@admin.com and password 'admin12'
  12. Create a new super admin account and delete admin@admin.com in postfixadmin
  13. Create new domain in postfixadmin
  14. Create new email address in postfixadmin
  15. Login to your new email at https://mail.[YOURDOMAIN].com
  16. [Static IP only, recommended] contact ISP and configure reverse lookup so your IP points to [YOURDOMAIN].com
  17. [Recommended] Create DNS record TXT for host mail._domainkey, and value in cat /var/lib/rspamd/dkim/mail.pub. Starting with v=DKIM1; and ending at last quote ". Remove spaces and quotes.
  18. [Recommended] Create DNS record TXT for host _dmarc with value "v=DMARC1; p=none; adkim=r; aspf=r;" without quotes.

Q&A

  • Do I need a static IP?
    Answer: No. Most companies will accept email from a dynamic IP mail server, but a few companies (e.g. craigslist email proxy) do not accept email from mail servers using dynamic IP.
  • Will AAT work with a dynamic IP?
    Answer: Yes, AAT works with a dynamic IP. You may need to do additional configuring of AAT through SSH to set up a cronjob that auto-updates the A record of your domain with your domain host. Many hosts offer a script for this. Some companies (e.g. craigslist email proxy) do not accept email from mail servers using dynamic IP.
  • Do I need to set up Reverse DNS for my IP?
    In an effort to curb abuse of email servers few companies (e.g. craigslist email proxy) do not accept email from computers with dynamically assigned IP addresses, or hosts that have generic DNS setup. This limits the amount of spam inflicted upon their users.
    Answer: You may be using a dynamic IP, and the IP address does not have the correct Reverse DNS configured. Contact your ISP and tell them you want a static IP if you don't already have one, and ask them to update the reverse DNS for your IP so it points to [YOURDOMAIN].com
  • Why am I receiving an email with the subject Undelivered Mail Returned to Sender
    The email reads: 554 5.7.1 <yourhost.yourdomain.com[10.10.10.1]>: Client host rejected: please_use_SMTP_relay_of_your_isp,_or_setup_non-generic_dns.
    Answer: You may be using a dynamic IP, and the IP address does not have the correct Reverse DNS configured. Contact your ISP and tell them you want a static IP, and ask them to update the reverse DNS for your IP so it points to [YOURDOMAIN].com
